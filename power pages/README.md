## IoT Panel Dashboard — Quick Start

A React 19 + Vite SPA deployed as a Power Pages code site. See [`iot-panel-dashboard/README.md`](iot-panel-dashboard/README.md) for the full guide.

### Prerequisites

| Tool | Version | Install |
|------|---------|---------|
| Node.js | 18+ | https://nodejs.org |
| PAC CLI | latest | `dotnet tool install --global Microsoft.PowerApps.CLI.Tool` |

### PAC CLI Commands

```powershell
# Authenticate (run once per session)
pac auth create --environment "https://YOUR_ORG_NAME.crm.dynamics.com"

# Check which environment is active
pac auth who

# List Power Pages sites in the environment
pac pages list

# Build and deploy the code site
cd "power pages\iot-panel-dashboard"
npm run build
pac pages upload-code-site --rootPath "."

# From the repo root (alternative)
pac pages upload-code-site --rootPath "power pages\iot-panel-dashboard"
```

> **Important:** Always use `pac pages upload-code-site` — never `pac pages upload`.  
> `pac pages upload` is for portal-studio sites and will corrupt code site metadata.

### Local Development

```powershell
cd "power pages\iot-panel-dashboard"
npm install       # first time only
npm run dev       # starts at http://localhost:5173
npm run build     # production build → dist/
```

## Key Conventions

- Always use `--modelVersion 2` for traditional portal (non-code-site) PAC commands
- Code sites use `pac pages upload-code-site` (no `--modelVersion` needed)
- `powerpages.config.json` at the project root defines `siteName` and `compiledPath`
- The `dist/` folder is generated by `npm run build` and uploaded to Power Pages
- Never commit `dist/` or `node_modules/` — both are gitignored

## Table Permissions

Configure table permissions in the **Power Pages admin center** or via PAC CLI:

```powershell
pac pages download --path "power pages" --webSiteId <WEBSITE_GUID> --modelVersion 2
```

See `.github/instructions/power-pages.instructions.md` for full conventions.
